📌 Desafio de Programação Orientada a Objetos em Java

Este projeto é um desafio prático de Programação Orientada a Objetos (POO) em Java, desenvolvido como parte do treinamento em desenvolvimento Java pela plataforma Digital Innovation One. O objetivo é aplicar e reforçar os principais pilares da POO — abstração, encapsulamento, herança e polimorfismo — em um projeto estruturado.

📌 Funcionalidades Principais

✔ Criação de objetos para representar cursos, mentorias e bootcamps ✔ Associação entre desenvolvedores (Devs) e conteúdos do bootcamp ✔ Lógica para inscrição de Devs em Bootcamps ✔ Registro e cálculo de progresso em cursos/mentorias

🧠 Visão Geral

O projeto simula um sistema de Bootcamp onde diferentes entidades como Bootcamp, Curso, Mentoria e Dev (desenvolvedor/participante) são modeladas como classes Java com relacionamentos entre si.

O foco não é apenas fazer o código “funcionar”, mas organizar o projeto em pacotes e classes que representem corretamente o domínio do problema, aplicando boas práticas de POO.

🚀 Tecnologias Utilizadas

✨ Java (JDK 11 ou superior)

Estruturas e conceitos de POO:

Classes e Objetos

Atributos e Métodos

Encapsulamento (getters/setters)

Herança e Polimorfismo

Interfaces e abstração

Coleções (List, Set)

Manipulação de datas (LocalDate)

📌 Conceitos de POO Aplicados

O projeto encapsula conceitos fundamentais da Programação Orientada a Objetos:

Conceito Descrição Abstração Separar o que um objeto faz do modo como faz. Encapsulamento Esconder dados internos e expor apenas o necessário. Herança Reutilizar código entre classes relacionadas. Polimorfismo Tratar objetos de classes diferentes via interface comum.

Esses conceitos são aplicados criando classes que representam o domínio de um bootcamp (cursos, mentorias, desenvolvedores, etc.).
